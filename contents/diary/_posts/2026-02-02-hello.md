## 再開に際して
　`Jekyll` + `github pages` でこのサイトは構成されている。新しく買ったミニPCでセットアップして `git clone` しても動かないという所から始まり、一応そのことについても書いておこうと思います。また5年後にきっと困ると思うので。

　大まかに分けて `build` と `deploy` で困りました。

### build - minima/sass
　これは主に `sass` 絡みであり、標準で採用している `minima` テーマの `scss` ファイルの記法が最新ではないために起こるビルドエラーでした。提供されている公式の `migrator` で直る所と直らない所があり、手動でガチャガチャしました。本家が対応してくれたら、それでオーバーライドしたい。

　明示的な `@import` 機能として `@use` が推奨されたり、 `lighten()` などのグローバル関数及びグローバル変数の使用が制限され `xxx.lighten()` のように明示することを要求するようになったわけで、私は賛成です。とはいえ、久しぶりにひとまず動かそうと思った時に、これやらされたのでちょっとげんなりしましたが、まあ修正は簡単だったので良いとしましょう。

### deploy
　これは主に `github action` の話です。これまで使っていたのは、`jekyll Docs V4.1.1` にて記述があるような、以下のようなものでした（ちょっと違うのでもう少しバージョンが低いDocsだったと思われる）

```yaml
name: Build and deploy Jekyll site to GitHub Pages

on:
  push:
    branches:
      - master

jobs:
  github-pages:
    runs-on: ubuntu-16.04
    steps:
      - uses: actions/checkout@v2
      - uses: helaili/jekyll-action@2.0.1
        env:
          JEKYLL_PAT: ${{ secrets.JEKYLL_PAT }}
```
https://jekyllrb-ja.github.io/docs/continuous-integration/github-actions/

　しかし、上記だとエラーが出て、エラー内容は `ruby` やら `bundle` のバージョンが合わないとかなんとかで、調整してエラー対応の方法はさっぱり分かりませんでした。

　そこで、以下の方の記述を発見しまして、無事解決しました。

　https://ryomo.github.io/notes/jekyll-github-pages

　この記事によると、どうやら `github pages` でのデプロイ方法として、わざわざ `Github Action` をセッティングから選択して、公式推奨の `jekyll github action` テンプレートを採用する必要があるということでした。これは、`jekyll Docs V4.4.1` に記載されていることでもあって、このバージョンを信頼して良さそうです。

```
Setting up the ActionPermalink

1. Go to the Settings tab on your repository.
    1. Click Pages under Code and automation.
    2. Change Source under Build and deployment from Deploy from a branch to GitHub Actions.

2. Go to the Actions tab on your repository.
    1. Start a New workflow and search for Jekyll.
    2. Click Configure under the Jekyll workflow (not GitHub Pages Jekyll workflow).
    3. Review the change and click Commit changes.
```
https://jekyllrb.com/docs/continuous-integration/github-actions/

　初めから `jekyll` の最新 Docs を見ておけばすぐに解決出来たようです。英語だからと忌避するのはエンジニアの向いてなさを実感します。

　ともかく、デプロイ問題もこれで解決しました。

## ようやく環境が整いましたので
　本題というか、5年ぶりに『逝く道』の環境セットアップをした理由とか書いていけたら良いなと思います。

　この5年の間に、社会復帰をしつつ、日常生活の整えを重点的に行ってきたように思います。特別何かに取り組んでいたということは無く、ただ「どのように生きるべきか」と日々問い続ける生活でしたし、これからも変わらないでしょう。しかし、一つだけ重要な変化が最近あります。

　**元気になってきた** ことです。

　退職して無職になってから、精神科に一度も通わなかったため、明確な診断として宣言出来ませんが、私は「鬱病」を患いました。退職タイミングからですので、鬱病自体は6年の付き合いになるでしょうか。その鬱病との付き合いも続く中で、どのように生きるのかを問う日々、最近少しずつ前向きに物事を捉えられるようになってきたと自認しています。

　特に昨年2025年、飲食店で接客業（ホール担当）をしているのですが、常連のお客様から「君の接客はとても良いね」「これでも飲んで頑張って」など評価や応援を頂いた一年でした。「良いお年を」の挨拶もしましたし、「今年もよろしくお願いします」の挨拶も常連さんにはしました。少なくとも、前職で、IT職で、能力も人格も何も褒められたものでは無かったという事実から、5年かかり、社会で評価していただけるような「何か」を獲得出来つつあるのだと驚いた一年でもありました。

　特別お金を稼ぎたいわけではないため、フリーターで十分だし、鬱病のこともあるしと、週4で働かせてもらっていますが、貯金も出来ました。2025年は株式投資にも挑戦して、少しプラスが出たりもして、経済的にも気持ち程度の余裕が生まれています。

　また、常連さんとの付き合いで、参加している講義での学びも2年ほど続いており、勉強の日々もあります。

　少しずつ人生の軌道が作れてきたように思うわけです。

　元気が出てくると、新しいことや、ほったらかしにしていた雑務やら何やらに手を付ける元気も出てくるものです。ええ、兼ねてから私は **「小説」を書いてみたい** と思っており、これは5年間ずっと思っていましたが、そろそろしっかり手を付けようと思ったのです。

　そのために、この『逝く道』を再始動させることにしました。

## 小説投稿

- 小説投稿サイトのUIよりも、私が自作した `Novel UI` の方が綺麗だと思ってしまったこと。
- 小説投稿サイトはそれぞれターゲット層（利用ユーザー層）が異なっており、 **ユーザーを考慮した作風** を暗黙的に要求されているに私が感じてしまうこと。
- 自分のペースで静かに続けていきたい、ということ。

　などが挙げられます。

　あと、サブスクっぽい感じで取り組んでみたいという思いもあります。具体的には「週次」で何か関係していることを書いて発信し、「月次」でまた発信するようなことです。こうした活動には `Fanbox` , `Cien` などがありますが、小説に適しているかは微妙です。加えて、金銭を取れるだけのあるいは応援して頂けるだけのコンテンツを安定供給出来るのかどうかについて、自分事ながら断言が難しい。

　しかし、この個人サイトなら、余計な事を考える必要がありません。ここで一年ぐらい継続して取り組めたなら、それを実績としてFanclub運用を視野に入れれば良いだろうと思っています。この活動で少しでもお金を稼げたなら、いつかフリーターを辞められる日も来るかもしれません。

　とはいえ、何よりも「小説」というメディア形式に自分も参画してみたいというのが一番の欲求です。

　**社会とどう接続して関わるかについての議論は、二の次で良い** のです。

　**小説執筆を自分との接続手法の一つとして採用していきたい** と思っていることが本意です。
